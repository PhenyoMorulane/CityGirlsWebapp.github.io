Internet Protocol

  • Internet protocols are sets of rules governing data transmission and sharing online. These ensure devices communicate effectively. Key protocols include
    1. TCP/IP (Transmission Control Protocol/Internet Protocol): Foundational protocol suite for data transport, addressing, and routing.
    2. HTTP (Hypertext Transfer Protocol): Used for sending web documents. HTTPS secures online transactions.
    3. SMTP (Simple Mail Transfer Protocol): Manages email sending and receiving between servers.
    4. POP3 (Post Office Protocol version 3): Retrieves email from servers to local devices.
    5. IMAP (Internet Message Access Protocol): Allows remote email access with server-based message management.
    6. FTP (File Transfer Protocol): Facilitates file transfers between client and server.
    7. DNS (Domain Name System): Converts domain names to IP addresses for user-friendly web access.
    8. SNMP (Simple Network Management Protocol): Manages and monitors network devices.
    9. DHCP (Dynamic Host Configuration Protocol): Automatically assigns IP addresses and network details.
    10. ICMP (Internet Control Message Protocol): Conveys error and operational info about network conditions.

Web Application Architecture, Client & Server Side Engineering.

Web Application Architecture

Web application architecture establishes the connections between applications, middleware systems, and databases to enable seamless collaboration among multiple applications. When a user inputs a URL and initiates a request, the browser contacts the website's server to retrieve the requested page. The vital element is the code interpreted by the browser. This code guides the browser's responses to various inputs. Thus, web application architecture encompasses all components and external interactions, creating a comprehensive software system. Efficiency and specific objectives are core considerations in its design. As the predominant method of global network communication, it serves all devices and applications. It addresses concerns like size, efficiency, resilience, and security.

Client-side Engineering

Client-side engineering involves developing web application components that operate within the user's web browser. These components handle user interface design, interactions, and presentation layer control.Client-side developers use coding to create appealing, functional online applications and dynamic websites. They manage everything visible or interactive on a website, such as homepages, shopping pages, and slideshows. This work occurs on the user's device. Often known as 'front-end' programmers, they manage the user-facing aspects of web pages. Front-end developers use scripting languages like JavaScript, HTML, and CSS to construct client-side code.

Server-side Engineering

Server-side engineering involves developing web application components that operate on the server, managing business logic, data storage, and client interactions. Server-side developers create and maintain the code that facilitates data exchange between the website and servers. Referred to as backend developers, they ensure the proper functioning of application servers behind the scenes. These professionals are skilled in server-side languages like Python, SQL, Ruby on Rails, PHP, and Java. They handle the functionality that enables websites to interact with databases and deliver services to users.
